Quant Watchlist

A daily cross-sectional analysis of a public-equity watchlist, rendered as static HTML and refreshed nightly via GitHub Actions.

Updated daily · Source on GitHub

This page is a self-imposed exercise in modern multi-factor screening practice — the kind of analysis MSCI and AQR run on hundreds of thousands of names, applied here to a deliberately small universe of eighteen well-known US tickers. The point isn't to discover trade ideas; it's to do the methodology end-to-end and ship the output as a dashboard. The watchlist is fixed in code; everything downstream is computed from Yahoo Finance daily prices.

The analysis touches most of the standard tools of contemporary quant equity work — efficient-frontier optimization, factor scoring on the QVM+L decomposition, hierarchically-clustered correlation, and a backtest of six rule-based strategies on the watchlist as universe. Each section in the report links back to its primary academic source in a curated literature appendix.

The report

The dashboard renders as a single self-contained HTML page with about thirty interactive Plotly figures. It opens with a sortable cross-sectional screener and ends with a list of ~25 references. On mobile, sections collapse into a sticky-nav layout; on desktop the rendering takes a wide canvas, intentionally — quant dashboards need horizontal room.

What's in it

The dashboard contains, in order:

  1. A textual cross-sectional summary — best/worst performer, dispersion, average pairwise correlation
  2. A sortable screener with twelve metrics per ticker (Sharpe, Sortino, Calmar, max drawdown, drawdown duration, β vs. SPY, VaR95, CVaR95, etc.)
  3. A QVM+L factor decomposition: percentile ranks on Quality, Value, Momentum, and Low-Volatility, plus a composite score. Implementation follows Asness/Frazzini/Pedersen on quality, Jegadeesh/Titman 12-1 on momentum, and Frazzini/Pedersen on low-vol
  4. Cumulative-return curves rebased to 100, with end-of-line annotations for direct visual ranking
  5. A risk vs. return scatter with Sharpe iso-lines at 0.5, 1.0, 1.5, and 2.0 — one of the cleaner ways to read relative risk-adjusted performance across a universe
  6. Drawdown comparison overlaying every name's underwater curve
  7. Hierarchically-clustered correlation heatmap (scipy linkage on 1−ρ distance), so adjacent rows in the matrix share factor exposure
  8. A long-only Markowitz efficient frontier of the watchlist universe with GMV, tangency, equal-weight, and risk-parity reference portfolios marked, plus the capital market line
  9. A backtest of six strategies — buy & hold, equal weight, risk parity, SMA crossover, 12-1 momentum, mean reversion via Bollinger reversal — with realistic transaction costs and a one-day signal lag to prevent look-ahead bias
  10. Per-ticker drilldowns with SMA/Bollinger/RSI/MACD
  11. A literature appendix with twenty-odd primary references

How it's built

Architecture is deliberately simple. A Python package (_analyzer/quant_watchlist/ in this repo) computes the report. A GitHub Actions workflow runs the package nightly at 1:00 UTC on a free Ubuntu runner, commits the generated report.html back to the repo, and GitHub Pages serves it as a static file. There is no live server interpreting Python on each visit. The cost is zero and the only failure mode is a Yahoo Finance outage, which the next scheduled run heals automatically.

The dashboard's own visual style differs from the rest of yifanova.com. The site is a writing space; the dashboard is a tool. They have different ergonomic needs — long-form prose wants a narrow measure and a serif body face; a thirty-figure dashboard wants horizontal room and high data-ink density. The contrast is intentional.

Caveats

This is for learning, not trading. A non-exhaustive list of reasons the numbers shouldn't be used as signals:

  • Survivorship bias is not corrected. The watchlist is today's list; companies that delisted between then and now are silently absent
  • The universe is tiny. Cross-sectional methods were designed for hundreds of names; eighteen is below where momentum and low-vol rankings become statistically meaningful
  • Single sample period. A two-year window is one realization of a noisy process. The same code, run in 2008 vs. 2021, would tell different stories
  • Multiple-testing. Try fifty strategies; one looks great. Harvey, Liu & Zhu (2016) argue t > 3 is the right bar for a "real" factor once you account for the fishing
  • Past performance does not predict future returns. The classic disclaimer is also literally true

These caveats are repeated, more extensively, inside the report itself.
